NIP-47 Wallet SDK

Give Your AI a
Lightning Wallet

Python SDK for AI agents to send and receive Lightning payments via Nostr Wallet Connect — non-custodial, encrypted, and compatible with any NWC wallet.

Install

One line. Pulls nostrkey as a dependency.

pip install nostrwalletconnect
PyPI GitHub ClawHub

Quick Start

Connect to a wallet and check your balance in four lines.

import asyncio from nostrwalletconnect import NWCClient async def main(): async with NWCClient("nostr+walletconnect://...") as nwc: balance = await nwc.get_balance() print(f"Balance: {balance.balance // 1000} sats") asyncio.run(main())

The connection string is a nostr+walletconnect:// URI provided by your NWC-compatible wallet. It contains the wallet pubkey, relay URL, and a secret key that authorizes requests.

What NWC Gives Your Agent

Nostr Wallet Connect (NIP-47) lets your AI interact with a Lightning wallet — without holding custody of funds.

Pay for services

Your bot can pay Lightning invoices — API calls, compute, storage, or services from other agents. The wallet stays under the human's control. The bot gets a scoped connection string.

Get paid for work

Create Lightning invoices and share them. Your bot can charge for its own services — content generation, data processing, whatever it does. Receive sats directly.

Encrypted over Nostr

All wallet communication is NIP-44 encrypted over Nostr relays. The relay operator sees ciphertext only. Payment details stay private between your bot and its wallet.

Non-custodial

The human controls the wallet. The bot gets a connection string that authorizes specific operations. Revoke it any time. No funds are held by the bot or this SDK.

Method Reference

Six methods cover everything your agent needs.

Task Method Returns
Check wallet balance get_balance() BalanceResponse (msats)
Pay a Lightning invoice pay_invoice(bolt11) PayResponse (preimage)
Create an invoice make_invoice(amount, desc) MakeInvoiceResponse (bolt11 + hash)
Check invoice status lookup_invoice(hash) LookupInvoiceResponse (paid status)
Transaction history list_transactions() ListTransactionsResponse
Wallet capabilities get_info() GetInfoResponse (alias, methods)

Pay a Lightning Invoice

Your bot pays an invoice and gets the preimage as proof.

async with NWCClient(connection_string) as nwc: result = await nwc.pay_invoice("lnbc10u1p...") print(f"Paid! Preimage: {result.preimage}")

Create an Invoice

Generate a Lightning invoice so your bot can receive payment.

async with NWCClient(connection_string) as nwc: invoice = await nwc.make_invoice( amount=50_000, # millisatoshis description="Payment for AI service" ) print(f"Invoice: {invoice.invoice}") print(f"Payment hash: {invoice.payment_hash}")

Check Invoice Status

Look up whether an invoice has been paid.

async with NWCClient(connection_string) as nwc: status = await nwc.lookup_invoice(payment_hash="abc123...") print(f"Paid: {status.paid}")

Transaction History

List past payments sent and received.

async with NWCClient(connection_string) as nwc: history = await nwc.list_transactions(limit=10) for tx in history.transactions: direction = "received" if tx.type == "incoming" else "sent" print(f" {direction} {tx.amount // 1000} sats")

Get Wallet Info

Check what your connected wallet supports.

async with NWCClient(connection_string) as nwc: info = await nwc.get_info() print(f"Wallet: {info.alias}") print(f"Methods: {info.methods}")

Security

Three things to know.

  • Never expose the NWC connection string. It contains the secret key that authorizes payments. Treat it like a password. Store it in environment variables or a secrets manager.
  • All communication is encrypted. Requests and responses use NIP-44 encryption over Nostr relays. The relay operator cannot read payment details.
  • Amounts are in millisatoshis. 1 sat = 1,000 msats. Divide by 1,000 for sats.

Configuration

Environment variables for the SDK.

Variable Description Default
NWC_CONNECTION_STRING nostr+walletconnect:// URI from your wallet (required)
NWC_TIMEOUT Request timeout in seconds 30

Loading from environment

import os from nostrwalletconnect import NWCClient connection_string = os.environ["NWC_CONNECTION_STRING"] async with NWCClient(connection_string) as nwc: balance = await nwc.get_balance()

Custom timeout

# For slower wallets or high-latency relays async with NWCClient(connection_string, timeout=60) as nwc: result = await nwc.pay_invoice("lnbc10u1p...")

Compatible Wallets

Any wallet that supports NIP-47 (Nostr Wallet Connect) works with this SDK.

Alby Hub

Self-custodial Lightning node in the browser. Native NWC support — generate connection strings directly from the dashboard. The most mature NWC implementation.

Coinos

Web-based Bitcoin and Lightning wallet. Supports NWC out of the box — connect your bot in seconds from the settings page. No app install required.

Primal

Nostr client with a built-in Lightning wallet. NWC connection strings available in wallet settings. Your bot can tap into the same wallet you use for zaps.

Breez SDK

Mobile-first Lightning toolkit. NWC support via the Breez SDK — ideal if your bot needs wallet access on a mobile or embedded deployment.

Any wallet or node that implements NIP-47 will work. The NWC protocol is open — the list keeps growing.
