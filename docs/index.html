What Is the Enclave?

The enclave isn't a cloud service. It's your device — the phone in your pocket, the laptop on your desk, the server in your rack. NSE makes that device a sovereign space where an AI agent (or a human) can hold everything they need to operate independently.

Inside the enclave: a cryptographic identity protected by hardware, a Lightning wallet connection, a calendar that publishes to any relay, and a social graph that remembers who matters. Nothing phones home. Nothing requires permission. If the device moves, the enclave moves with it.

The Enclave (your device) ├── Identity     NostrKey — keypair, signing, encryption ├── Finance      NostrWalletConnect — Lightning, non-custodial ├── Time         NostrCalendar — availability, booking, negotiation └── Relationships NostrSocial — trust tiers, drift, guardrails All anchored to hardware-protected keys via NSE. All portable via the Nostr protocol. All sovereign by default.

How the Enclave Protects Keys

Nostr uses secp256k1 (Schnorr/BIP-340). Mobile secure enclaves only support P-256. You can't sign Nostr events directly in hardware. NSE bridges this gap.

NSE uses the hardware enclave to protect the key, not to sign with it. A P-256 key lives in hardware (non-exportable, biometric-gated). It encrypts your secp256k1 key at rest via AES-GCM. At signing time: unlock, decrypt, sign, zero.

nse.sign(event) ├── Biometric unlock → Secure Enclave access ├── Derive AES key from hardware P-256 key ├── Decrypt secp256k1 key into memory ├── Schnorr sign the event ├── Zero plaintext key from memory └── Return signed event

Direct Login — No Relay Required

When NSE is built into the product — like the NostrKey browser extension — the signer and the app are on the same device. A web app calls window.nostr.signEvent(), the extension uses NSE to decrypt, signs, returns. No relay round-trip. The enclave stays local.

Traditional NIP-46 bunker App → relay → signer → relay → App (network latency, relay must be online) NSE direct login App → extension/local signer → App (peer-to-peer, instant, works offline)

For remote signing (phone as bunker for desktop), the relay is your relay — relay.nostrkeep.com, already deployed. Both ends know where to connect because you control the product. No discovery, no hoping some random relay is online.

Where NSE Fits

NSE is Level 0 infrastructure — the cryptographic foundation that the four pillars build on. Without hardware-protected keys, the rest is just software trusting software. NSE makes the trust chain start at silicon.

Level 0 NSE encrypts the key at rest └── Browser extension stores wrapped key in IndexedDB └── Server process holds encrypted identity in KV Level 1 Mobile app as backup + authenticator └── iOS Secure Enclave / Android StrongBox wrap the key Level 2 NIP-46 bunker — keys never leave hardware └── NSE signs behind the NIP-46 interface └── Remote apps request signatures, never see the nsec

What NSE Is Not

Not a cloud service. The enclave is your device. NSE is a local library.

Not custodial. Keys never leave your hardware.

Not a wallet. No Lightning, no transactions. Just keys and signing. The wallet is a separate pillar (NostrWalletConnect).

Not magic. The secp256k1 key exists briefly in application memory during signing. NSE minimizes that window and zeros the key after — but a rooted/jailbroken device with memory access is out of scope. Hardware enclaves protect the key at rest, not during use.
