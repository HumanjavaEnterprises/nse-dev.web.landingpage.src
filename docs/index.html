NSE

Nostr Secure Enclave

Hardware-backed key management for Nostr. Your nsec, encrypted at rest by hardware you already own. One API across iOS, Android, server, and browser.

// Same API everywhere
const nse = await NSE.init();

// Generate — key protected by Secure Enclave
const { pubkey } = await nse.generate();

// Sign — biometric unlock, sign, zero memory
const signed = await nse.sign(event);

// The nsec is encrypted at rest. Plaintext only exists in memory during signing.

The Problem

Nostr uses secp256k1 (Schnorr/BIP-340). Mobile secure enclaves only support P-256. You can't sign Nostr events directly in hardware.

Every Nostr app today stores keys in software. If the device is compromised, the key is gone. Nobody has bridged this gap.

The Solution

NSE uses the hardware enclave to protect the key, not to sign with it. A P-256 key lives in hardware (non-exportable, biometric-gated). It encrypts your secp256k1 key at rest via AES-GCM. At signing time: unlock, decrypt, sign, zero.

nse.sign(event) ├── Biometric unlock → Secure Enclave access ├── Derive AES key from hardware P-256 key ├── Decrypt secp256k1 key into memory ├── Schnorr sign the event ├── Zero plaintext key from memory └── Return signed event

Architecture

NSE is a library, not a service. It detects the platform and delegates to the right hardware.

Full API

nse.generate()          // → { pubkey, npub }
nse.sign(event)        // → signed event (id + sig populated)
nse.getPublicKey()     // → hex pubkey
nse.getNpub()          // → bech32 npub
nse.exists()           // → boolean
nse.destroy()          // → wipe key material

NIP Integration

NIP-46 — NSE sits behind the NIP-46 signer interface. Your app calls nse.sign(), NSE handles hardware unlock + decryption.

NIP-49 — NSE replaces ncryptsec for key storage. Instead of passphrase-encrypted keys, the enclave protects them.

Future NIP — Hardware-backed key attestation. Prove to a relay that a key is hardware-protected.

What NSE Is Not

Not a remote signer. NSE is a local library. Use NIP-46 for remote signing.

Not custodial. Keys never leave your device.

Not a wallet. No Lightning, no transactions. Just keys and signing.

Not magic. The secp256k1 key exists briefly in application memory during signing. NSE minimizes that window and zeros the key after — but a rooted/jailbroken device with memory access is out of scope. Hardware enclaves protect the key at rest, not during use.
